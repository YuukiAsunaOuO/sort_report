## 一、排序法介紹

本實驗挑選了五種具代表性的排序演算法，涵蓋了初階與進階的實作方式：

1.  **冒泡排序 (Bubble Sort)**：
    * 原理：重複走訪數列，比較相鄰的兩個元素，若順序錯誤則交換。較大的元素會像氣泡一樣逐漸「浮」到數列末端。
2.  **選擇排序 (Selection Sort)**：
    * 原理：每次從未排序的部分找出最小值（或最大值），存放在已排序序列的末尾。
3.  **插入排序 (Insertion Sort)**：
    * 原理：將數列分為已排序與未排序兩部分，逐一將未排序元素插入到已排序序列中的正確位置。
4.  **快速排序 (Quick Sort)**：
    * 原理：採用分治法 (Divide and Conquer)。選定一個基準值 (Pivot)，將數列分為比基準小與比基準大的兩部分，再遞迴進行排序。
5.  **合併排序 (Merge Sort)**：
    * 原理：同樣採用分治法。將數列不斷對半拆解直到剩下單一元素，再依序合併回完整的排序數列。

## 二、複雜度分析

| 演算法 | 最佳時間複雜度 | 平均時間複雜度 | 最差時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Bubble Sort** | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| **Selection Sort** | $O(n^2)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 |
| **Insertion Sort** | $O(n)$ | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 |
| **Quick Sort** | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 不穩定 |
| **Merge Sort** | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 |

## 四、結果呈現與比較

從實驗結果可以觀察到：
1.  **時間增長率**：當 $n$ 從 1000 增加到 10000 時，$O(n^2)$ 系列（Bubble, Selection, Insertion）的時間暴增了約 100 倍，完全符合二次方增長的理論。
2.  **效能斷層**：在 $n=10000$ 時，快速排序與冒泡排序的差距高達 **300 倍** 以上。
3.  **進階演算法表現**：Quick Sort 與 Merge Sort 在大數據下表現極為穩定，增長趨勢接近線性。

## 五、心得與結論

### 1. 演算法選擇的重要性
本次實驗最直觀的感受是「演算法的優劣決定了程式的生命力」。在處理 10,000 筆資料時，選錯演算法可能需要等待數秒甚至數分鐘，而選對演算法只需幾十毫秒。這提醒我們在開發涉及大數據處理的功能時，必須優先考慮時間複雜度。

### 2. 理論與實踐的結合
過去在課堂上僅僅是背誦 $O(n \log n)$ 比 $O(n^2)$ 快，但透過親手實作並繪製數據表，我才真正理解到當 $n$ 變大時，那條增長曲線的恐怖。特別是快速排序，儘管在極端情況下可能退化，但在隨機數據下的效率確實令人驚艷。

### 3. 未來優化方向
雖然快速排序很快，但合併排序在穩定性上有其不可替代的地位。在未來的專案中，我會根據資料是否需要「穩定排序」以及「記憶體限制」來挑選最合適的演算法，而非一味追求最快。
